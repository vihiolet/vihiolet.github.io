프로젝트 소개

소개

최근 높은 수요를 보이는 윤리적인 제품을 모아 이용자가 간단하게 투표하는 방식으로 후기를 작성하는 사이트
 이용자에게는 제품의 후기를 보고 윤리적인 소비에 대한 장벽을 낮추고 판매자에게는 홍보 수단이 될 수 있다.

특징

새로운 개념의 검색점이라는 키워드로 메뉴에 상관없이 모든 제품을 검색할 수 있다.
 예) '친환경' 이라는 검색점으로 검색하면 검색점이 친환경으로 등록된 음식이나 생활 관련 제품이 모두 조회된다.

사이트 구성

사용자 : 로그인/회원가입, 마이페이지, 제품 목록,
제품 상세보기, 후기 투표하기

관리자 : 로그인, 제품 설정, 관리자 설정, 검색점 설정,
후기 관리, 내 정보 관리

프로젝트의 MVC 구조

MVC 패턴을 사용하여 프로젝트를 만들었습니다.
아래 이미지는 관리자 페이지 중 검색점 조회 페이지를 Model, Controller, View로 구분한 흐름도입니다.

  • POINT 1. ajax 사용

    해당 페이지 : (관리자) 제품 설정, 검색점 설정, 후기 관리 / (사용자) 내가 투표한 후기
    기능 : 페이지 전환 없이 삭제 또는 수정, 삭제 처리

  • POINT 2. 비밀번호 SHA256 암호화

    해당 페이지 : (관리자) 관리자 등록, 로그인 / (사용자) 회원 가입, 로그인
    기능 : 비밀번호가 DB에 저장될 때 SHA-256 해시 알고리즘과 Salt(무작위 문자열)를 이용하여 비밀번호 암호화

  • POINT 3. inner join

    해당페이지 : (사용자) 제품 뷰 페이지, 마이페이지
    기능 : 해당 제품의 후기 중 후기 제목의 각 투표 수와 가장 많은 선택을 받은 후기 제목 조회 등

  • POINT 4. 다양한 js, jQuery 사용

    해당페이지 : (관리자) 검색점 설정 외 다수 페이지 / (사용자) 제품 뷰 페이지, id 중복체크 외 다수 페이지
    기능 : 다중 체크, 팝업, 데이터를 추가할 input 생성, 입력한 값 서로 일치 확인, 요소의 최대값에 따라 style 추가 등

  • POINT 5. chart.js 라이브러리 사용

    해당페이지 : (관리자) 데시보드
    기능 : 각 메뉴 별 등록된 상품 개수 출력

  • POINT 6. AWS로 프로젝트 배포

    해당페이지 : (관리자) 데시보드
